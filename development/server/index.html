            服务器部署

            服务器初始化

            在 Vultr 上新建一个 1 CPU, 1 GB RAM, 25 GB SSD、位于日本的服务器，选择相应的 ssh key；安装完成之后会提示需要重启。

            重启后登录服务器，安装 Docker 和 Docker Compose:

            apt-get update && apt-get install -y ca-certificates curl gnupg lsb-release
mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
apt-get update && apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

            防火墙的相关设置

            由于网关也是一个 Docker 容器，为了实现反向代理它需要能够访问宿主机上的端口。在宿主机上，由于网关是一个由 Docker Compose 部署的服务，它的地址通常是 172.x.x.x，但在有些服务器上本地端口不能被 172 开头的 IP 访问，所以需要手动进行防火墙设置。为了检视网关的具体地址，可以在网关开始运行后执行

            docker network ls

            找到名称为 gateway_default 的网络后，再执行

            docker network inspect gateway_default

            找到如下图所示的部分

                    "IPAM": {
            "Driver": "default",
            "Options": null,
            "Config": [
                {
                    "Subnet": "172.18.0.0/16",
                    "Gateway": "172.18.0.1"
                }
            ]
        },

            其中 Subnet 后面的地址就是网关的地址。在 Ubuntu 系统上，防火墙的管理工具是 ufw，可以执行

            ufw allow from 172.18.0.0/16

            来允许网关访问其他服务。
